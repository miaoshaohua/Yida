# AI虚拟试衣间 V1.0 - Product Requirement Document

## Overview

- **Summary**: AI虚拟试衣间是一款基于响应式设计的虚拟试衣Web应用，为用户提供便捷的在线试衣体验。用户只需上传一张个人照片和一张衣服图片，系统即可调用AI虚拟试衣API生成穿搭效果图。同时配套后台管理系统，支持管理员对用户、订单、试衣记录等核心数据的增删改查操作。
- **Purpose**: 解决传统网购无法试穿的痛点，提供效果最好的免费虚拟试衣工具，支持任意图片，无需注册即可使用。
- **Target Users**: 
  - 前端：普通消费者（60%）、网购达人（40%）
  - 后台：系统管理员、运营人员

## Goals

### 前端目标
- 提供简单易用的图片上传和试衣功能
- 实现移动端优先的响应式设计
- 验证核心功能与付费意愿
- 试衣完成率 >50%，平均等待时长 <3分钟，畅用版转化率 >3%

### 后台目标
- 实现系统数据的增删改查基础管理
- 提供数据概览和趋势分析
- 支持订单管理和退款处理
- 记录管理员操作日志，确保系统安全

## Non-Goals (Out of Scope)

### 前端非目标
- ❌ 高清原图下载（技术成本高，V2.0规划）
- ❌ 云端历史记录同步（需后端开发，V1.5规划）
- ❌ 专业版/白标嵌入（需API支持批量，V2.0规划）
- ❌ 3D建模功能
- ❌ 微信小程序开发

### 后台非目标
- ❌ 多租户管理
- ❌ 复杂数据分析和BI报表
- ❌ 自动化营销功能

## Background & Context

- **API依赖**: 使用FASHN VTON API（`https://miaoshaohua--fashn-vton-api-fastapi-app.modal.run`）进行虚拟试衣生成
- **UI参考**: 前端参考唯品会VIP.com设计风格，后台采用企业级管理系统风格
- **技术选型**: 前端React + Vite + Tailwind CSS；后台React + Ant Design + NestJS + PostgreSQL
- **商业模式**: 免费版每日3次 + 畅用版¥9.9/月或¥49/年（无限次+优先队列）

## Functional Requirements

### 前端功能要求

- **FR-1**: 用户可以上传人物照片和衣服图片
- **FR-2**: 用户可以选择服装类型（上衣/下装/连体）
- **FR-3**: 用户可以点击"开始试衣"生成效果图
- **FR-4**: 用户可以查看、下载和分享试衣结果
- **FR-5**: 用户可以查看和管理试衣历史记录（localStorage，最多10条）
- **FR-6**: 系统提供每日3次免费试衣，超过后引导付费
- **FR-7**: 用户可以购买畅用版获得无限次试衣
- **FR-8**: 系统提供加载状态和分阶段提示
- **FR-9**: 系统提供友好的错误提示
- **FR-10**: 应用支持移动端、平板端、桌面端响应式布局

### 后台功能要求

- **FR-11**: 管理员可以登录/退出后台系统
- **FR-12**: 管理员可以查看数据概览（今日试衣次数、付费用户数、总收入等）
- **FR-13**: 管理员可以查看和管理用户列表
- **FR-14**: 管理员可以查看和管理订单列表（包括退款处理）
- **FR-15**: 管理员可以查看和管理试衣记录
- **FR-16**: 管理员可以配置系统参数（API配置、价格配置等）
- **FR-17**: 系统记录管理员操作日志
- **FR-18**: 系统支持超级管理员和普通管理员两种角色权限

## Non-Functional Requirements

### 前端非功能要求

- **NFR-1**: 页面加载时间 <2秒
- **NFR-2**: API超时设置为1200秒（20分钟）
- **NFR-3**: 支持图片格式：JPG、PNG、WEBP，单张不超过10MB
- **NFR-4**: 按钮和可点击元素最小点击区域44x44px
- **NFR-5**: 支持现代浏览器（Chrome、Firefox、Safari、Edge最新版本）
- **NFR-6**: 历史记录只存缩略图，单条<100KB

### 后台非功能要求

- **NFR-7**: 后台页面响应时间 <1秒
- **NFR-8**: JWT token有效期2小时，支持token刷新
- **NFR-9**: 密码使用bcrypt加密存储
- **NFR-10**: 登录失败5次后锁定30分钟
- **NFR-11**: 操作日志保留期限6个月
- **NFR-12**: 支持SQL注入、XSS、CSRF防护

## Constraints

### 技术约束

- **前端框架**: React 18+
- **构建工具**: Vite 5+
- **样式方案**: Tailwind CSS 3+
- **后台UI**: Ant Design 5+
- **后端框架**: NestJS
- **数据库**: PostgreSQL
- **ORM**: Prisma
- **外部API**: FASHN VTON API（`https://miaoshaohua--fashn-vton-api-fastapi-app.modal.run`）

### 业务约束

- **开发周期**: 前端3周，后台5周
- **预算限制**: API调用成本控制
- **合规要求**: 隐私保护提示、用户数据安全

### 依赖约束

- FASHN VTON API服务可用性
- 微信支付/支付宝支付接口可用性

## Assumptions

- FASHN VTON API服务稳定可用，CORS配置正确
- 用户使用现代浏览器访问应用
- 支付服务（微信支付/支付宝）接入可行
- 后端服务器和数据库资源充足
- 用户理解并接受首次请求可能需要2-5分钟的等待时间

## Acceptance Criteria

### AC-1: 图片上传功能
- **Given**: 用户在试衣页面
- **When**: 用户点击上传区域或拖拽图片到上传区域
- **Then**: 系统显示图片缩略图，支持重新上传
- **Verification**: `human-judgment`
- **Notes**: 支持JPG、PNG、WEBP格式，单张不超过10MB

### AC-2: 试衣功能
- **Given**: 用户已上传人物照片和衣服图片
- **When**: 用户点击"开始试衣"按钮
- **Then**: 系统显示加载状态，生成并展示试衣效果图
- **Verification**: `programmatic`
- **Notes**: 免费用户每日3次限制

### AC-3: 结果展示与下载
- **Given**: 试衣效果图已生成
- **When**: 用户点击下载按钮
- **Then**: 图片保存到用户本地设备
- **Verification**: `human-judgment`

### AC-4: 历史记录功能
- **Given**: 用户有试衣历史记录
- **When**: 用户查看历史记录
- **Then**: 显示缩略图列表，最多10条，支持清空
- **Verification**: `programmatic`

### AC-5: 付费引导
- **Given**: 免费用户今日次数已用完
- **When**: 用户再次点击"开始试衣"
- **Then**: 显示付费升级引导弹窗
- **Verification**: `human-judgment`

### AC-6: 响应式布局
- **Given**: 用户在不同设备访问应用
- **When**: 窗口大小变化或使用不同设备
- **Then**: 页面布局自适应，移动端上下堆叠，桌面端并排
- **Verification**: `human-judgment`

### AC-7: 后台管理员登录
- **Given**: 管理员访问后台登录页
- **When**: 输入正确的用户名和密码
- **Then**: 登录成功，跳转到仪表板
- **Verification**: `programmatic`

### AC-8: 数据概览
- **Given**: 管理员已登录后台
- **When**: 访问仪表板页面
- **Then**: 显示今日试衣次数、付费用户数、总收入等核心数据
- **Verification**: `human-judgment`

### AC-9: 用户管理
- **Given**: 管理员在用户管理页面
- **When**: 进行搜索、查看详情、启用/禁用操作
- **Then**: 操作成功，数据更新
- **Verification**: `programmatic`

### AC-10: 订单管理
- **Given**: 管理员在订单管理页面
- **When**: 查看订单详情、进行退款操作
- **Then**: 操作成功，订单状态更新
- **Verification**: `programmatic`

### AC-11: 试衣记录管理
- **Given**: 管理员在试衣记录页面
- **When**: 查看记录详情、删除记录
- **Then**: 操作成功，记录更新
- **Verification**: `programmatic`

### AC-12: 系统配置
- **Given**: 超级管理员在系统配置页面
- **When**: 修改API配置、价格配置
- **Then**: 配置保存成功，系统使用新配置
- **Verification**: `programmatic`

### AC-13: 权限控制
- **Given**: 普通管理员登录
- **When**: 尝试访问管理员管理、系统配置、操作日志页面
- **Then**: 权限不足，无法访问
- **Verification**: `programmatic`

### AC-14: 操作日志
- **Given**: 管理员进行了敏感操作
- **When**: 查看操作日志
- **Then**: 显示操作人、操作时间、IP地址、操作详情
- **Verification**: `programmatic`
