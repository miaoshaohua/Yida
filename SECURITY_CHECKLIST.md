# Yida 生产环境安全检查清单

## 部署前检查

### 🔐 安全配置

- [ ] **JWT_SECRET** 已更换为强随机密钥（非默认值）
  ```bash
  openssl rand -base64 32
  ```

- [ ] **.env 文件** 已加入 `.gitignore`，不提交到仓库

- [ ] **Swagger 文档** 生产环境建议关闭或添加认证
  - 修改 `main.ts`：`if (process.env.NODE_ENV !== 'production') { SwaggerModule.setup(...) }`

- [ ] **CORS 配置** 已设置正确的生产域名（非 localhost）

- [ ] **数据库密码** 使用强密码，非默认 `postgres/postgres`

- [ ] **管理员默认密码** 生产环境已修改 `superadmin` 的初始密码

- [ ] **API 限流** 已配置接口请求频率限制（防刷）

- [ ] **敏感日志** 不输出 JWT token、密码等敏感信息到日志

### 🗂️ 代码质量

- [ ] **console.log** 调试日志已移除（生产环境使用 Winston 等日志库）

- [ ] **错误处理** 全局异常过滤器已配置，不暴露内部错误堆栈

- [ ] **TypeScript** 编译检查通过，无 `any` 类型滥用

- [ ] **依赖漏洞** 运行 `npm audit` 检查并修复已知漏洞

### 📦 依赖检查

- [ ] **package.json** 中无未使用的依赖

- [ ] **生产依赖** 使用固定版本号，不使用 `*` 或 `latest`

- [ ] **devDependencies** 不会部署到生产环境

### 🌐 网络配置

- [ ] **HTTPS** 已启用（Vercel/Railway 自动提供）

- [ ] **数据库** 不对外开放端口，仅内部访问

- [ ] **防火墙** 仅开放必要端口（443/80）

### 📊 监控与备份

- [ ] **日志收集** 已配置日志服务（如 Railway Logs / Vercel Logs）

- [ ] **错误追踪** 已接入 Sentry 或其他错误监控

- [ ] **数据库备份** Railway 自动备份已启用

- [ ] **健康检查** 后端已配置 `/health` 端点

### 📱 前端检查

- [ ] **API Base URL** 已配置为生产后端地址

- [ ] **敏感信息** 前端不暴露 API 密钥、JWT secret 等

- [ ] **Source Map** 生产环境不生成 `.map` 文件

- [ ] **压缩** 已启用 Gzip/Brotli 压缩
